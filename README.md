## Which APK Should I Download?

Each release provides multiple APK variants for different CPU architectures.

| Suffix | Architecture | Who should use this? |
|--------|-------------|----------------------|
| *(no suffix)* | Universal / Pure-Java | All devices — works everywhere |
| `-arm` | 32-bit ARM | Older Android phones (before ~2016) |
| `-arm64` ⭐ | 64-bit ARM | **Most modern Android phones — recommended** |
| `-x86` | 32-bit x86 | Older emulators |
| `-x64` | 64-bit x86 | Modern emulators and Intel Chromebooks |
| `-universal` | All architectures | Any device (largest file size) |

**Quick rule:** If unsure, always download the **`-arm64`** variant. It works on virtually all Android smartphones made since 2016.

### Manual Setup (If Buttons Don't Open Obtainium)

If the buttons don't work (e.g. on a desktop browser or if Obtainium isn't installed), add any app manually:

1. Open Obtainium → tap **"Add App"**
2. Paste the URL: `https://github.com/nOneCode4u/mixplorer-releases-google`
3. Scroll to **"Additional Options"** → find **"Filter Release Titles By (RegEx)"**
4. Enter the filter from the table below for the app you want
5. Enable **"Auto-select APK by architecture"**
6. Tap **"Add"**

| App | Filter Release Titles By |
|-----|--------------------------|
| MiXplorer | `MiXplorer-v` |
| MiX Archive | `MiX_Archive-v` |
| MiX Codecs | `MiX_Codecs-v` |
| MiX Encrypt | `MiX_Encrypt-v` |
| MiX Image | `MiX_Image-v` |
| MiX PDF | `MiX_PDF-v` |
| MiX Tagger | `MiX_Tagger-v` |

### Installing a Specific CPU Variant

By default, Obtainium picks the right APK for your device automatically. If you want to force a specific variant, in **"Additional Options"**:

- Disable **"Auto-select APK by architecture"**
- Set **"APK Filter (RegEx)"** to the value from the table below

| Variant | APK Filter (RegEx) |
|---------|-------------------|
| ARM 64-bit (arm64) | `arm64` |
| ARM 32-bit (arm) | `-arm\.apk` |
| x86 64-bit (x64) | `x64` |
| x86 32-bit (x86) | `-x86\.apk` |
| Universal | `universal` |
| Pure-Java (no suffix) | `_B[0-9]+\.apk$` |

## Why This Repository Exists

MiXplorer and its add-ons are distributed via the [XDA forum thread](https://xdaforums.com/t/app-2-2-mixplorer-v6-x-released-fully-featured-file-manager.1523691/) and a private Google Drive folder. While the apps are excellent, their distribution has practical limitations:

- The XDA thread requires manual checking for new versions.
- The in-app updater for MiXplorer does not clean up downloaded APK files, wasting storage over time.
- There is no single location listing all add-ons and all APK architecture variants together.
- Tools like Obtainium cannot natively track XDA threads.

This repository solves all of these problems by automatically mirroring the official releases to GitHub every day.
